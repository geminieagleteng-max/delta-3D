# 📅 DELTA FORCE: 3D 戰術訓練基地 - 工作日誌

本文檔記錄了本專案的所有功能迭代、優化與修復歷程。

---

## 📝 歷史開發日誌與修改紀錄

### 🚀 階段 1：基礎系統修復與裝備邏輯調整
*   **個人倉庫 (Stash) 重新分組與過濾**：
    *   重構個人倉庫清單，過濾掉數量為 `0` 的冗餘項目，只顯示玩家實際擁有的物品。
    *   將倉庫物品依據類型清晰地分為四大分組：`槍械武器`、`戰術裝具與配件`、`戰術消耗品`、`撤離戰利品`，方便玩家閱讀。
*   **修復武器裝備 (Loadout) 衝突 Bug**：
    *   修正了玩家在個人倉庫裝備非預設武器時，將武器「型號」（如 `'ak47'`）直接作為實體 ID 傳入，導致裝備欄無法順暢穿戴與穿戴衝突的問題。
    *   現在會動態查找個人倉庫中該武器實體的唯一 `uid`（如 `'item_jjnlrxkuj'`）並進行穿戴，徹底修復「明明有武器卻帶不上」的 Bug。
*   **搜刮時滑鼠重複鎖定 Bug 修復**：
    *   修正了玩家在遊戲內走近加密箱搜刮物資時，一點擊彈出的搜刮面板，滑鼠又被立刻鎖定回螢幕中央，導致無法移動游標拖曳物資的 Bug。
    *   現在在開啟搜刮面板（Loot Modal）時，會暫時完全銷毀/暫停 `PointerLockControls` 的監聽，確保搜刮狀態下能流暢解鎖滑鼠。

---

### 🚀 階段 2：行動端適配與管理員功能引入
*   **防止行動端雙指控制導致網頁縮放**：
    *   在 `index.html` 中優化 `viewport` meta 宣告，限制最大縮放比例並停用使用者縮放。
    *   在 `index.css` 的全域樣式中加入 `touch-action: none !important` 與 `-webkit-text-size-adjust: none`，停用一切瀏覽器預設手勢。
    *   在 `App.jsx` 中監聽並攔截多指觸碰事件（當觸碰點數 `e.touches.length > 1` 時），徹底防止手機端雙指旋轉視角時意外縮放整個遊戲網頁。
*   **管理員特權帳號功能 (初步)**：
    *   為特定的開發者/管理者帳號開啟特權（鎖定 Delta 幣為 `99999999`，不扣錢）。
    *   設計管理員專用作弊快捷鍵：`O` 鍵跳過波次、`P` 鍵補滿血量、`K` 鍵一鍵獲勝通關。
    *   實戰中增加左側管理員專屬 CRT 綠色作弊控制台，方便在觸控裝置或滑鼠解鎖時手動點按操作。

---

### 🚀 階段 3：行動端 HUD 排版適配與視角靈敏度優化
*   **解決行動端橫屏下的 HUD 重疊與遮擋**：
    *   **戰術背包面板重定位**：將戰術背包 HUD 面板 `.hud-backpack-panel` 在手機端移到左上角雷達正下方 (`top: 110px; left: 15px; transform-origin: top left`)，釋放左下角虛擬搖桿，防止阻礙腳步移動。
    *   **底部 HUD 卡片極限壓縮與置中**：將底部狀態卡片群組 `.hud-bottom` 改為水平置中對齊底部，並將每個卡片寬度由 `220px` 壓縮至 `108px`，精簡間距與字體大小，縮小收攏在搖桿與按鈕中間的空檔，避免遮擋左右操作。
    *   **輔助按鈕分流防重疊**：
        *   暫停按鈕 (`.mobile-pause-btn`) 移至左上角雷達右側 (`left: 195px; top: 12px; right: auto`)，與補血和扔雷並列。
        *   切槍按鈕 (`.btn-switch`) 移至右上角 (`right: 15px; top: 12px;`)，避開任務資訊欄。
    *   **右下動作按鍵防擠出**：將動作按鈕容器 `.mobile-buttons-container` 和開火按鈕 `.btn-fire` 的底邊上抬 `10px`，防止貼底而被手機底部系統手勢條遮擋。
*   **行動端視角旋轉靈敏度提升**：
    *   將觸控移動視角的平滑靈敏度 `sensitivity` 由 `0.0035` 提高一倍至 `0.007`，大幅改善行動裝置上滑動轉向靈活性。

### 🚀 階段 4：管理員帳號持久化與本機遷移
*   **管理員特權持久化**：
    *   移除了在程式碼中硬編碼判定英文帳號名稱 `"distant star"` 的動態 runtime Override 邏輯，避免任何新註冊相同命名的帳號自動升格管理員。
    *   直接更新 KVdb 雲端資料庫中 `"distant star"`（暱稱 `"遙星"`）該筆帳號的 JSON 記錄，將 `"isAdmin": true` 與 `"coins": 99999999` 持久化寫入。
    *   在本機的 `getAccounts` 函數中，加入針對 **帳號名為 `"distant star"`、帳號名為 `"遙星"`，或暱稱為 `"遙星"`** 的一次性本機帳號升級與 LocalStorage 資料遷移，確保已登入的使用者在不重登或重新整理網頁時，均能順利轉換為 persistent 的管理員帳號屬性。

### 🚀 階段 5：管理員遊戲金鑰與雙重驗證 (2FA) 登入系統
*   **限制常規登入渠道**：
    *   在 `src/utils/account.js` 中對常規的 `loginAccount` 密碼登入進行限制。若使用者試圖輸入 `"distant star"`，系統會直接拒絕並提示切換至「金鑰登入 KEY」分頁進行雙重驗證。
*   **新增遊戲金鑰與動態驗證碼雙重驗證**：
    *   在主登入畫面新增第三個 Tab：**「金鑰登入 KEY」**。
    *   要求輸入**遊戲金鑰 (`DELTA-ADMIN-9999-STAR`)** + **密碼** + **6 位動態口令 Token** 三重欄位。
    *   實作基於確定性時間序列（每 30 秒變更）的 OTP 生產演算法 `getOtp`。並設定前後 30 秒容差範圍以保證時差容錯。
*   **設計本機電子密碼器 (Secure Token Widget)**：
    *   在金鑰登入介面下方嵌入一個戰術 CRT 綠色發光的 **「電子密碼安全器」**。
    *   實時顯示當下有效之 6 位數安全密碼，並提供一個隨時間流逝遞減的 30 秒倒數計時進度條，兼顧安全要求與極佳的使用者測試體驗。
*   **地面敵軍 AI 撞牆卡死防範機制**：
    *   在 `Enemy` 的 `useFrame` 更新中，加入前置與後置位置座標比較（以 Vector3 distance 計算實際移動距離）。
    *   設定卡牆判定條件（若移動速度低於預期速度的 15%），並設計一個 1.2 秒的卡牆計時器。
    *   一旦敵軍卡牆超過 1.2 秒，強制為其進行「路徑重劃與狀態轉移」（在 `COVERS` 中隨機尋找新掩體，或將狀態切換至衝鋒狀態），使敵軍能智慧地繞過牆壁或掩體障礙物，恢復攻擊威脅。

### 🚀 階段 6：地下通道隨機事件與空間扭曲、交戰區域實作
*   **地鐵長廊「8號出口」玩法優化**：
    *   實作波次肅清前進機制：每波敵人消滅後，玩家必須前進至通道盡頭，此時出口指示牌遞減（8 ➔ 7 ➔ ... ➔ 1）。
    *   在出口 1 肅清後，鐵捲門（Shutter Door）自動向上升起開啟，玩家走上樓梯撤離獲勝。
*   **暫停回復狀態保持與作弊跳關傳送**：
    *   修正了玩家在暫停後返回遊戲時狀態遺失並重設回起點的 Bug，現在可正確保留進度與 3D 座標。
    *   管理員在地下通道執行跳關（ CRT 控制台或快捷鍵 `O`）時，程式會自動將玩家傳送至通道盡頭附近 (`z = -105` 米)，免去冗長的手動跑圖測試。
*   **地下通道專屬物理碰撞與倉庫圖片美化**：
    *   實作 `FACILITY_COLLIDERS` 陣列，對自動販賣機、垃圾桶、軍事掩體箱等加入物理碰撞體積，杜絕穿牆。
    *   對個人倉庫 Stash 的物品進行 PNG 去背，並使物品圖片渲染尺寸完美對齊所佔用的背包網格格數。
*   **進圖隨機事件系統 (Random Events)**：
    *   **電力中斷 (Blackout)**：日光燈熄滅，玩家主武器上自動開啟錐形戰術手電筒（SpotLight）提供視野。
    *   **化學濃霧 (Dense Fog)**：在地鐵長廊中生成綠色厚霧。
    *   **紅色警報 (Red Alert)**：閃爍應急紅光並伴隨低沉警報蜂鳴。
    *   **空間扭曲 (Space Warp)**：
        *   實作 3D 空間 Sine 正弦扭曲投影公式，將直線地鐵分段渲染為彎曲 S 型長廊。
        *   將長廊靜態結構物、指示牌、販賣機、垃圾桶等掩體及日光燈物理位置依投影公式動態移位與轉動。
        *   動態扭曲 `PlayerController` 與 `Enemy` 的移動邊界及 AABB 碰撞盒陣列，使物理碰撞體與彎曲的視覺模型完全對齊。
    *   **交戰區域 (Combat Zone)**：
        *   在地鐵內生成 2 名 DELTA 友軍特種部隊，採用青色防具與藍色頭盔，頭頂顯示「DELTA 友軍」血條與姓名。
        *   友軍採用動態 AI：自主搜尋並交火最近的敵軍；敵軍亦會對友軍與玩家進行最近目標優先度評估與開火。
        *   設定友軍傷害免疫判定（玩家對友軍射擊不造成傷害與 Hitmarker 標記，僅產生擊中火花效果）。
        *   將友軍過濾排除在長廊波次肅清的計數條件外，保證通關條件的正確性。
